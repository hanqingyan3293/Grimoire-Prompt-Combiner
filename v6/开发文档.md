# 魔导书 Grimoire v6.0 — 开发文档

## 一、环境准备

### 必需环境
- **Node.js** >= 18 (推荐 22+)
- **npm** >= 9
- **Windows** 10/11 (macOS/Linux 需调整打包配置)

### 克隆与安装
`ash
cd v6
npm install
`

### 开发模式
`ash
npm run dev
`
这会同时启动 Vite 开发服务器 (端口 5173) 和 Electron 窗口。

## 四、架构设计

### 4.1 进程模型

`
┌──────────────────────────────────┐
│         Main Process             │
│  ┌────────────┐ ┌─────────────┐ │
│  │  Window    │ │  Database    │ │
│  │  Manager   │ │  (sql.js)    │ │
│  └────────────┘ └─────────────┘ │
│  ┌────────────────────────────┐ │
│  │  IPC Handlers              │ │
│  │  tags / presets / history  │ │
│  │  settings / images         │ │
│  └────────────────────────────┘ │
└───────────┬──────────────────────┘
            │ IPC (invoke/handle)
┌───────────▼──────────────────────┐
│       Preload (Bridge)           │
│  contextBridge.exposeInMainWorld │
└───────────┬──────────────────────┘
            │ window.electronAPI
┌───────────▼──────────────────────┐
│      Renderer Process            │
│  ┌────────────────────────────┐ │
│  │  React App                 │ │
│  │  ┌──────┐ ┌─────────────┐ │ │
│  │  │Views │ │ Zustand     │ │ │
│  │  │      │ │ Stores      │ │ │
│  │  └──────┘ └─────────────┘ │ │
│  └────────────────────────────┘ │
└──────────────────────────────────┘
`

### 4.2 数据流

1. **Renderer** 通过 window.electronAPI.xxx.invoke() 发起请求
2. **Preload** 将调用转发为 ipcRenderer.invoke('channel', ...args)
3. **Main Process** 的 IPC handler 处理请求，操作 SQLite 数据库
4. 结果原路返回 Renderer，更新 Zustand Store
5. React 组件响应 Store 变化，重新渲染

### 4.3 数据库 Schema

`sql
categories (id TEXT PK, name TEXT, sort_order INT, created_at TEXT)
subcategories (id TEXT PK, category_id TEXT FK, name TEXT, sort_order INT, created_at TEXT)
tags (id TEXT PK, subcategory_id TEXT FK, en TEXT, zh TEXT, sort_order INT, source TEXT, created_at TEXT)
favorites (id INT PK, tag_id TEXT FK UNIQUE, created_at TEXT)
tag_usage (tag_id TEXT PK FK, count INT, last_used TEXT)
presets (id TEXT PK, name TEXT, data TEXT JSON, created_at TEXT, updated_at TEXT)
history (id TEXT PK, prompt TEXT, data TEXT JSON, created_at TEXT)
snapshots (id TEXT PK, name TEXT, data TEXT JSON, created_at TEXT)
settings (key TEXT PK, value TEXT)
image_refs (id INT PK, file_path TEXT, metadata TEXT JSON, created_at TEXT)
migration_status (version INT PK, migrated_at TEXT, source_version TEXT)
`

### 4.4 撤销/重做设计

使用双栈模式：
- undoStack — 每次操作前将当前状态快照压入
- edoStack — 执行撤销时将当前状态压入
- maxUndo — 最大保留 50 步

## 六、关键模块说明

### 6.1 AIPanel.tsx
src/renderer/components/panels/AIPanel.tsx

- 双 Tab 设计：Chat 和 Vision
- Chat Tab：调用 /chat/completions，使用 OpenAI 兼容格式
- Vision Tab：读取图片 → base64 → /chat/completions (vision 格式) → 解析标签
- API Key / Endpoint 从 Zustand settings.store 读取，无硬编码
- 支持 AbortController 中断请求

### 6.2 prompts.store.ts
src/renderer/stores/prompts.store.ts

核心状态：positive[], 
egative[], undoStack[], edoStack[]

关键方法：
- ddPositive/removePositive — 正向标签增删，自动记录 undo
- updateWeight — 修改标签权重
- getFormattedPrompt(lang) — 格式化输出（含权重语法 (tag:1.2)）
- getFullPrompt() — 完整提示词文本（正向 + 负向）
- deduplicate() — 按 tag.id 去重

### 6.3 database.ts
src/main/database.ts

- 使用 sql.js 的 WASM 版本，无需原生依赖
- 数据库文件保存在 pp.getPath('userData')/grimoire.db
- WAL 模式 + 外键约束 + 5 秒 busy timeout
- saveDatabase() 将内存数据库序列化写入磁盘

## 八、常见问题

### 构建后样式丢失
确保 postcss.config.cjs 无 BOM 头，否则 PostCSS 配置加载失败。

### ASAR 缺少 dist 文件夹
检查 package.json 的 uild.files 是否包含 "dist/**/*"。

### 便携版无法保存数据
便携版默认用 process.env.GRIMOIRE_PORTABLE 判断模式。确保启动脚本设置了此环境变量。

### AI 请求 401 错误
检查 Settings 中 API Key 和 Endpoint 是否正确配置。注意 Endpoint 应以 /v1 结尾（OpenAI 兼容格式）。
