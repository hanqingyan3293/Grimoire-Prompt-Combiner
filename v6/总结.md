# 魔导书 Grimoire v6.0 — 项目总结

## 项目概述

魔导书 Grimoire 是一款 **AI 绘画提示词工作站**，基于 Electron + React + TypeScript 构建的跨平台桌面应用。灵感来源于知名开源项目 [Chatbox](https://github.com/chatboxai/chatbox)，在标签化提示词管理的基础上，集成了 OpenAI 兼容的 AI 聊天与视觉识别功能。

## 技术架构

| 层级 | 技术 | 用途 |
|------|------|------|
| 桌面框架 | Electron 42 | 窗口管理、系统集成、本地文件访问 |
| 前端渲染 | React 18 + Vite 8 + Tailwind CSS 4 | UI 渲染与交互 |
| 状态管理 | Zustand 5 | 全局状态（提示词、标签库、设置） |
| 本地存储 | sql.js (SQLite WASM) | 标签库、预设、历史记录、设置 |
| 构建工具 | TypeScript 6 + Vite + electron-builder | 编译与打包 |
| AI 接口 | OpenAI 兼容 API (chat + vision) | AI 聊天与图片分析 |

## 核心功能

1. **标签化提示词管理** — 三级分类（类别 → 子类 → 标签），支持搜索、收藏、频率统计、拖拽添加、权重调节
2. **双提示词面板** — 正向/负向提示词独立管理，支持撤销/重做（50步）、去重、随机打乱
3. **AI 聊天** — 对话式 AI，支持切换模型、自定义 System Prompt
4. **AI 识图** — 拖拽/点击上传图片，AI 分析并生成标签，可一键添加到提示词或保存到标签库
5. **预设管理** — 保存/加载提示词组合为预设
6. **历史记录** — 自动保存提示词组合历史
7. **7 套主题** — Neon / Clean / Gold / Midnight / Sakura / Forest / Sunset
8. **中英双语** — 界面和标签双语支持
9. **数据导入导出** — JSON 格式标签库和预设导入导出

## 数据存储

使用 **sql.js** (SQLite 编译为 WebAssembly) 进行本地数据持久化：

- categories / subcategories / 	ags — 三级标签库
- avorites / 	ag_usage — 收藏和使用频率统计
- presets / history / snapshots — 预设和历史
- settings — 键值对存储（主题、语言、AI 配置等）
- image_refs — 图片引用和元数据

数据文件存储在 Electron userData 目录下的 grimoire.db。
