|------|--------|------|

| randomMin | 3 | 随机标签最小数量 |
| randomMax | 8 | 随机标签最大数量 |
| defaultWeight | 1.0 | 默认标签权重 |
| showQualityTags | true | 是否自动添加 masterpiece, best_quality |
| outputFormat | en | 输出格式: en/zh/both |
| defaultNegative | 长字符串 | 默认负面词 |
| categoryColors | true | 是否显示分类色标 |
| aiProvider | wd14 | AI 提供者: wd14/grok/openai/custom |
| aiEndpoint | (空) | API 端点 |
| aiApiKey | (空) | API Key |
| aiModel | grok-vision | 模型名 |
| workspace | default | 工作区: default/compact/wide-canvas/two-row |

Prompts Store 关键方法:
- addPositive(tag), removePositive(tagId)
- addNegative(tag), removeNegative(tagId)
- updateWeight(tagId, weight, list)

- undo(), redo() — 50 步历史
- deduplicate()

## 十一、常见问题和注意事项

### 构建报错修复
1. 确保 C:\Program Files\nodejs 在 PATH 中
2. node_modules 损坏: 删除后重新 npm install
3. RollDown 解析错误: App.tsx 中避免 for...of + 分号组合
4. PowerShell 写文件: 必须用 UTF8Encoding(false) 无 BOM

### 数据库 (SQLite)
- 使用 sql.js (SQLite WASM, 内存运行)
- 数据库文件: %APPDATA%/grimoire/grimoire.db
- 通过 saveDatabase() 手动持久化 (每次写操作后调用)

### 浮动面板机制
- panel-manager.ts 管理所有浮动窗口生命周期
- 每个浮动面板是独立 BrowserWindow
- 窗口 URL (dev): http://localhost:5173/#/float/{panelId}
- 窗口 URL (prod): file://...#/float/{panelId}
- main.tsx 检测 hash 路由决定渲染 FloatPanel 还是 App
- 浮动窗口关闭时自动回到停靠区 (panel:docked IPC)
- 拖回主窗口右侧 >60px 重叠, 自动停靠
- 靠近主窗口边缘 20px, 自动吸附

### IPC 通信模式
- ipcMain.handle + ipcRenderer.invoke: 双向异步 (数据操作)
- ipcMain.on + ipcRenderer.send: 单向推送 (菜单命令)
- 菜单到渲染进程: mainWindow.webContents.send(menu:xxx)
- 渲染进程 relay: api.send(trigger:xxx)
