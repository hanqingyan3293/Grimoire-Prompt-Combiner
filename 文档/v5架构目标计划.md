# 魔导书 v5 — 架构目标计划

> 创建时间：2026-06-11
> 项目仓库：https://github.com/hanqingyan3293/Grimoire-Prompt-Combiner

---

## 一、项目愿景

从"AI 绘画提示词组合器"升级为 **AI 绘画全流程工作台**。
不仅要组合提示词，更要成为 AI 绘画创作者的核心工作站：管理标签库、处理参考图、连接 AI 模型、扩展插件生态。

---

## 二、技术选型

| 维度 | v4.x (旧) | v5 (新) |
|------|-----------|---------|
| 前端框架 | 原生 HTML/CSS/JS | React 18 + TypeScript |
| 样式方案 | 手写 CSS (~26KB) | Tailwind CSS + shadcn/ui |
| 后端 | Python Flask | Node.js (Electron 主进程) |
| 桌面壳 | 浏览器 | Electron |
| 数据存储 | JSON 文件 | SQLite (better-sqlite3) |
| 状态管理 | 全局变量 | zustand |
| 构建工具 | 无 | Vite + electron-builder |

## 五、开发路线图

| 阶段 | 内容 | 预计产物 |
|------|------|----------|
| 1. 环境搭建 | Node.js + 项目脚手架 | 可运行的空 Electron 窗口 |
| 2. 数据层 | SQLite schema + 迁移脚本 | v4 数据成功导入 SQLite |

| 7. 高级功能 | 预设/历史/快照/导出 | 数据管理功能完 |
| 8. 图片面板 | 拖放/元数据/参考管理 | 图片工作流可用 |
| 9. 设置 + AI | 设置页 + AI 接口预留 | 配置面板可用 |

## 七、数据迁移策略

1. 读取 `v4.3/data/tags.json` → 解析 categories/subcategories/tags 结构
2. 读取 `v4.3/data/custom_tags.json` → 合并自定义标签
3. 生成唯一 ID（`{category_id}_{subcategory}_{en}` 哈希）
4. 写入 SQLite，标记 source='builtin' 或 'custom'
5. 预设/历史/快照文件 → 解析后写入对应表
6. 迁移完成后 v4.x 数据原封不动保留

## 八、关键假设

- 目标平台：Windows（未来可扩展 Mac/Linux）
- 用户自行配置 AI API Key（不内置任何密钥）
- 首版不实现：插件加载、云同步、标签图片预览
- 窗口默认 1400×900，支持自由缩放
- 所有用户数据存储在应用数据目录（AppData）
